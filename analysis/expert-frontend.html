Сайтът страда от критични проблеми с производителността: 36 MB неоптимизирано изображение, 1.1 MB CSS бъндъл, 1.7 MB+ JavaScript без defer/async, липса на Cache-Control хедъри и пълно отсъствие на security headers. Общото тегло на началната страница надвишава 38 MB, при препоръчителни <2 MB. Core Web Vitals категорично не се покриват.

3. Критични проблеми (подредени по тежест)
  • #1: Неоптимизирано изображение от 36 MB Файл: IMG_20260317_130800.jpg — 36,640,293 байта (34.9 MB). Директно от камера, без resize. Върху 3G връзка това са ~5 минути за зареждане.
    Решение: Resize до 1200px ширина + WebP формат = ~80-120 KB. Спестяване: ~36.5 MB.
  • #2: Fusion CSS бъндъл от 1.1 MB Файл: f5f57a7ea19e6305a89ba244190ad9e9.min.css — 1,131,516 байта (1.08 MB). Render-blocking CSS, парсира се преди първото рендериране.
    Решение: Critical CSS inline (~15 KB) + async зареждане на останалото. Purge неизползвани стилове (очаквано ~70% неизползвани). Спестяване: ~800 KB.
  • #3: 7 render-blocking JavaScript файла (1.71 MB общо) jquery.min.js (88 KB) + jquery-migrate (14 KB) + LayerSlider utils (123 KB) + LayerSlider core (277 KB) + LayerSlider transitions (33 KB) + LightSlider (16 KB) + Fusion scripts (596 KB) = 1,147 KB. Нито един няма async или defer атрибут. RevSlider добавя още 581 KB.
    Решение: defer на всички скриптове + елиминиране на 2 от 3 слайдера. Спестяване: ~800-1000 KB + 2-3s TBT.
  • #4: Пълна липса на Security Headers Липсват: Content-Security-Policy, Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy.
    Решение: Конфигуриране в .htaccess или nginx. Нулев performance cost.
  • #5: Липса на Cache-Control хедъри Нито HTML, нито статичните ресурси имат Cache-Control хедъри. Само Last-Modified присъства за изображенията. Браузърът не може ефективно да кешира нищо.
    Решение: Cache-Control: max-age=31536000 за статични assets, max-age=3600 за HTML.
  • #6: Mixed Content — HTTP линкове 4 незащитени HTTP URL-а: ws.naval-acad.bg, projects.nvna.eu, studentcouncil.naval-acad.bg, pp.naval-acad.bg. Adwise tracking pixel също е protocol-relative (//as.adwise.bg).
    Решение: Мигриране на всички линкове към HTTPS.
  • #7: Липса на lazy loading за изображения Нито едно изображение на началната страница няма loading="lazy". Всички 33+ изображения се зареждат едновременно.
    Решение: Добавяне на loading="lazy" за below-the-fold изображения. WordPress 6.x го поддържа нативно.
  • #8: HTML документ от 245 KB Началната страница е 244,738 байта (239 KB) некомпресирана, 33 KB gzip. Съдържа 8 inline <style> блока с page-specific CSS за 29+ различни page ID-та.
    Решение: Преместване на inline CSS във файлове, елиминиране на page-specific стилове.

Критични проблеми

  • 7 render-blocking скрипта общо 1,122 KB Нито един от основните скриптове няма defer или async атрибут. Браузърът спира рендирането докато не ги свали и изпълни всичките.
  • 3 слайдер библиотеки (LayerSlider + RevSlider + LightSlider) Общо 1,005 KB JavaScript само за слайдери. Един модерен слайдер (напр. Swiper.js) = ~40 KB minified+gzip.
  • jquery-migrate.min.js е ненужен Осигурява обратна съвместимост за jQuery <3.0 код. Ако плъгините са актуални, може да се премахне (14 KB).

  • Service Worker (Workbox 7.3.0) PWA поддръжка с prefetch стратегия за документи. Добра практика за повторни посещения.
  • Google Analytics 4 (gtag) Модерна имплементация, зарежда се асинхронно.

Inline CSS анализ

  • 8 inline <style> блока в <head> Съдържат page-specific CSS за 29+ различни page ID-та (8257, 8464, 8325, 8722, 8833, 8846, ...). Тези стилове се зареждат на ВСЯКА страница, дори когато са нужни само за една конкретна.
  • CSS Variables дефинирани правилно WordPress 6.x global styles (цветове, разстояния, сенки) са добре организирани. Но custom properties за Avada са дублирани.

CSS Стратегия — проблеми

  • Липса на Critical CSS Целият 1.1 MB CSS файл се зарежда синхронно в <head>. Трябва да се извлечат ~10-15 KB критични стилове за above-the-fold съдържание и да се inline-нат, а останалото да се зареди асинхронно.
  • Дублирани CSS ресурси за слайдери LayerSlider CSS (31 KB) + RevSlider CSS (57 KB) + LightSlider CSS (5 KB) = 93 KB за слайдери.
  • Font Awesome 4.7 (остаряла версия) Версия 4.7 е от 2017 г. Актуалната е 6.x. Алтернатива: SVG sprite с използваните икони (~2-5 KB вместо 30 KB CSS + 155 KB шрифтове).
  • Липса на CSS tree-shaking/purging Оценка: ~60-70% от стиловете в fusion CSS бъндъла са неизползвани на дадена страница. PurgeCSS/UnCSS може да свали размера до ~300-400 KB.

Системни проблеми

  • Липса на автоматична оптимизация при качване WordPress качва оригиналните файлове без компресия. Файл от 36 MB директно от камера показва, че няма плъгин за оптимизация (Imagify, ShortPixel, EWWW).

Service Worker (Workbox)

  • Workbox 7.3.0 е инсталиран PWA plugin регистрира Service Worker с conservative prefetch стратегия. Добра основа за offline caching.
  • Стратегията е твърде конservативна Само навигационни заявки се prefetch-ват. CSS/JS/шрифтове не се кешират от SW. Препоръка: CacheFirst за статични ресурси, StaleWhileRevalidate за API заявки.

CDN използване

  • Минимално CDN използване Само LightSlider се зарежда от CDN (jsdelivr.net). Всички останали ресурси идват от origin сървъра (sh-cache). Изображения се зареждат от 2 различни домейна: www.naval-acad.bg и naval-acad.nvna.eu, без preconnect hints.
    Препоръка: Cloudflare CDN за всички статични ресурси.
