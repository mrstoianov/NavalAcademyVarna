7. Производителност и бързодействие

Основното потребителско оплакване е бавното зареждане на страниците. Техническият анализ разкрива конкретните причини — натрупване на тежки плъгини, неоптимизирани ресурси и остарели подходи.

Основни фактори за бавно зареждане (по тежест)

1
Двоен слайдер — LayerSlider + Revolution Slider Зареждат се ЕДНОВРЕМЕННО два тежки слайдер плъгина. Revolution Slider се инициализира на ВСЯКА страница, включително вътрешни, където не се използва.
~800KB
2
Avada inline CSS ~20KB CSS код се вгражда директно в HTML-а на всяка страница. Включва стилове за 40+ други страници, които не се посещават. Този CSS не може да бъде кеширан от браузъра.
~20KB/стр.
3
Неоптимизирани изображения Всички изображения са в JPEG/PNG формат — без модерни WebP/AVIF. Логото е 136KB PNG вместо ~5KB SVG. Липсва lazy loading на повечето изображения.
+300-500KB
4
WPML overhead Плъгинът за многоезичност зарежда cookie management и UI компоненти на всяка страница.
~50-100KB
5
Fusion Builder DOM сложност Шаблонът генерира 5-6 нива вложени контейнери за всеки елемент, което забавя визуализирането.
DOM bloat
6
WordPress emoji система Зарежда detection скрипт и SVG набори за emoji — напълно ненужно за академичен сайт.
~8KB

9. Обобщение на идентифицираните проблеми

Критични

Производителност: Двоен слайдер (LayerSlider + Revolution Slider) добавя ~800KB ненужен код. Страниците тежат 3-5MB.
HTTP линкове: ws.naval-acad.bg, projects.nvna.eu, studentcouncil.naval-acad.bg използват незащитен HTTP протокол.
Неактуална английска версия: Новините на английски изостават с години.

Средни

Некеширан CSS: 20KB inline стилове на всяка страница — браузърът не може да ги кешира.
Неоптимизирани изображения: Без WebP, без lazy loading. Логото е 136KB PNG.
Разпръсната инфраструктура: 9+ отделни портала с различен дизайн и потребителско изживяване.
Достъпност: Непълна WCAG 2.1 имплементация.

Положителни практики (за запазване)

Декларация за достъпност до обществена информация
Банкова информация за студенти в долната навигация
Добре структурирана страница за специалности с филтри
Интеграция със социални мрежи (Facebook, X, Instagram, LinkedIn)
Service Worker за подобрена производителност при повторни посещения

11. Препоръки за подобрен сайт

Модерна платформа (Next.js): Вграден server-side rendering, автоматично code splitting, оптимизирано зареждане. 4-6 пъти по-бързо от текущия сайт.
Оптимизирани изображения: Автоматична конверсия в WebP/AVIF, responsive размери, вграден lazy loading. Логото като SVG (~5KB вместо 136KB).
Лек hero компонент: CSS-only или минимален слайдер (~40KB) вместо 800KB двоен слайдер.
Вградена многоезичност: Next.js i18n routing без нужда от тежък плъгин. Български и английски от старта.
Модерен CSS: Автоматично tree-shaking, минификация и кеширане. Само използваните стилове на всяка страница.
Чиста архитектура: Компонентна система с 2-3 нива вместо 5-6 нива wrapper елементи.
HTTPS навсякъде: Всички линкове през защитен протокол.
WCAG 2.1 достъпност: Пълна ARIA имплементация, keyboard navigation, screen reader поддръжка.
